                        RUTAS DE LECTURA RECOMENDADAS
================================================================================

RUTA 1: CLIENTE/STAKEHOLDER
----------------------------
Objetivo: Entender y aprobar la entrega

1. indice_documentacion.txt (este archivo) - 5 min
2. resumen_ejecutivo.txt - 15 min
3. control_cambios.txt (sección de aprobación) - 5 min
4. (Opcional) readme.txt (Instrucciones de Uso) - 10 min

Tiempo total: ~25-35 minutos

RUTA 2: DESARROLLADOR NUEVO
----------------------------
Objetivo: Incorporarse al proyecto

1. indice_documentacion.txt (este archivo) - 5 min
2. resumen_ejecutivo.txt - 15 min
3. readme.txt completo - 30 min
4. guia_versionado.txt - 20 min
5. control_cambios.txt - 15 min
6. Revisar código fuente - 60+ min

Tiempo total: ~2-3 horas

RUTA 3: USUARIO FINAL
---------------------
Objetivo: Usar el sistema

1. readme.txt (sección Instrucciones de Uso) - 10 min
2. Probar el sistema - 20 min

Tiempo total: ~30 minutos

RUTA 4: AUDITOR DE SEGURIDAD
-----------------------------
Objetivo: Auditar la seguridad

1. resumen_ejecutivo.txt - 15 min
2. readme.txt (sección Detalles Técnicos de Seguridad) - 20 min
3. control_cambios.txt completo - 30 min
4. Revisar código fuente - 120+ min
5. Ejecutar calcular_md5.py para verificar integridad - 5 min

Tiempo total: ~3+ horas

RUTA 5: DESARROLLADOR HACIENDO CAMBIOS
---------------------------------------
Objetivo: Crear una nueva versión

1. guia_versionado.txt completo - 20 min
2. control_cambios.txt (plantilla) - 5 min
3. Hacer los cambios en el código - variable
4. Ejecutar calcular_md5.py - 2 min
5. Actualizar readme.txt - 10 min
6. Actualizar control_cambios.txt - 15 min

Tiempo total: ~52+ minutos (más tiempo de desarrollo)

                        HERRAMIENTAS INCLUIDAS

CALCULAR_MD5.PY
---------------
Script de Python para calcular y verificar hashes MD5

FUNCIONES:
1. Generar reporte de MD5 de archivos actuales
   - Tabla formateada
   - Formato para documentación
   - Formato compacto

2. Verificar integridad contra versión 1.0.0
   - Compara hashes actuales vs documentados
   - Identifica archivos modificados
   - Identifica archivos faltantes

USO:
python calcular_md5.py
[Selecciona opción del menú]

CUÁNDO USAR:
- Al crear una nueva versión (para obtener MD5)
- Al recibir archivos (para verificar integridad)
- En auditorías de seguridad
- Al sospechar modificaciones no autorizadas

================================================================================
                        GLOSARIO DE TÉRMINOS
================================================================================

TÉRMINOS COMUNES EN LA DOCUMENTACIÓN:

MD5: Algoritmo de hash para verificación de integridad de archivos
SHA-256: Algoritmo criptográfico de hash de 256 bits
AES-256-CBC: Cifrado simétrico de 256 bits en modo CBC
RSA-2048: Cifrado asimétrico con claves de 2048 bits
HMAC: Código de autenticación de mensajes basado en hash
TLS/SSL: Protocolo de seguridad de la capa de transporte
SemVer: Versionado Semántico (MAJOR.MINOR.PATCH)
Rollback: Revertir a una versión anterior del software

Para más términos, consulte el Glosario en control_cambios.txt

================================================================================
                        CONVENCIONES DE DOCUMENTACIÓN
================================================================================

SÍMBOLOS USADOS:
✓ - Completado/Implementado/Correcto
□ - Pendiente/No implementado
❌ - Error/Faltante
⚠️  - Advertencia/Modificado
🔐 - Cifrado/Confidencialidad
🔨 - Hashing/Integridad
🔑 - Autenticación/Claves

FORMATO DE VERSIONES:
X.Y.Z donde:
- X = MAJOR (cambios incompatibles)
- Y = MINOR (nuevas funcionalidades compatibles)
- Z = PATCH (correcciones de bugs)

FORMATO DE FECHAS:
DD de Mes de AAAA
Ejemplo: 19 de Octubre de 2025

FORMATO DE MD5:
Siempre en MAYÚSCULAS, 32 caracteres hexadecimales
Ejemplo: 4AAB3B47E85B88C5F461E39AB63834A1

                        MANTENIMIENTO DE DOCUMENTACIÓN
================================================================================

CUANDO CREAR UNA NUEVA VERSIÓN:

1. Actualizar readme.txt:
   - Añadir nueva entrada en "Historial de Versiones"
   - Incluir MD5 de archivos modificados

2. Actualizar control_cambios.txt:
   - Añadir nueva entrada con todos los detalles
   - Documentar razones y impacto
   - Incluir MD5 de archivos

3. NO modificar:
   - guia_versionado.txt (solo si cambian procedimientos)
   - indice_documentacion.txt (solo si se añaden nuevos documentos)
   - resumen_ejecutivo.txt (crear nuevo resumen_ejecutivo_vX.Y.Z.txt para cada versión mayor)

4. Usar calcular_md5.py:
   - Para obtener MD5 automáticamente
   - Para verificar integridad
